# 🧪 Testcase Specification: NakhonFlood Alert & Recovery System
**รายการทดสอบระบบเตือนภัยล่วงหน้าน้ำท่วม และบริหารจัดการภัยพิบัติ จังหวัดนครศรีธรรมราช**

เอกสารฉบับนี้ระบุรายละเอียดของกรณีทดสอบ (Testcases) ทั้งหมดใน Sprint ปัจจุบัน เพื่อตรวจสอบความถูกต้องของฟังก์ชันงาน (Functional Testing), การแสดงผลบนอุปกรณ์พกพา (UI/UX Responsive Testing) และการซิงก์ข้อมูลคลาวด์เรียลไทม์ (Cloud Sync Testing)

---

## 🔐 มิติที่ 1: ระบบยืนยันตัวตน และสิทธิ์ผู้ใช้งาน (Authentication Module: TC-AUTH)

| รหัสเทสเคส | ชื่อรายการทดสอบ | ขั้นตอนการทดสอบ (Test Steps) | ผลลัพธ์ที่คาดหวัง (Expected Results) | สถานะ |
| :--- | :--- | :--- | :--- | :---: |
| **TC-AUTH-01** | เข้าสู่ระบบด้วย Google Account | 1. ไปที่หน้าล็อกอิน กดปุ่ม "เข้าสู่ระบบด้วย Google"<br>2. เลือกบัญชี Gmail ในหน้าต่าง Popup<br>3. ตรวจสอบสถานะมุมขวาบนของหน้าเว็บ | ล็อกอินสำเร็จ โลโก้และชื่อผู้ใช้เปลี่ยนเป็นชื่อบัญชี Google พร้อมบันทึก Session ลง LocalStorage | 🟢 Pass |
| **TC-AUTH-02** | เข้าสู่ระบบด้วย LINE Account | 1. กดปุ่ม "เข้าสู่ระบบด้วย LINE"<br>2. ยืนยันสิทธิ์ผ่าน LINE OIDC | ล็อกอินสำเร็จ ดึง Display Name จากโปรไฟล์ LINE มาแสดงผลได้อย่างถูกต้อง | 🟢 Pass |
| **TC-AUTH-03** | ออกจากระบบ (Sign Out) | 1. กดที่รูปโปรไฟล์มุมขวาบน<br>2. เลือกเมนู "ออกจากระบบ" | ล้างค่า Session และสิทธิ์การใช้งาน กลับสู่สถานะผู้เยี่ยมชม (Guest Citizen) ทันที | 🟢 Pass |

## 📱 มิติที่ 2: ส่วนต่อประสานผู้ใช้งาน และการตอบสนองบนมือถือ (UI/UX & Navigation: TC-NAV)

| รหัสเทสเคส | ชื่อรายการทดสอบ | ขั้นตอนการทดสอบ (Test Steps) | ผลลัพธ์ที่คาดหวัง (Expected Results) | สถานะ |
| :--- | :--- | :--- | :--- | :---: |
| **TC-NAV-01** | ตรวจสอบดีไซน์ Glassmorphism | 1. เปิดหน้าแรกบนเบราว์เซอร์<br>2. สังเกตพื้นหลังของการ์ดรายงานต่างๆ | พื้นหลังมีความกึ่งโปร่งใสแบบกระจกเงาไล่เฉดสี (Blur Backdrop Filter) สวยงามทันสมัย | 🟢 Pass |
| **TC-NAV-02** | Smart Navbar: ซ่อนเมื่อเลื่อนลง | 1. เปิดเว็บในขนาดหน้าจอมือถือ (375px)<br>2. ใช้นิ้วเลื่อนหน้าจอลงด้านล่างเกิน 50px | แถบสถานะสีส้มหดตัวหายไป และแถบโลโก้ใหญ่สไลด์ขึ้นแอบด้านบนสุด (`-translate-y-full`) ทันที | 🟢 Pass |
| **TC-NAV-03** | Smart Navbar: โชว์เมื่อเลื่อนขึ้น | 1. จากข้อ TC-NAV-02 ใช้นิ้วเลื่อนหน้าจอขึ้นเล็กน้อย | แถบเมนูกะทัดรัดสไลด์กลับลงมาอย่างนุ่มนวล ให้กดใช้งานได้โดยไม่ต้องเลื่อนไปสุดทาง | 🟢 Pass |
| **TC-NAV-04** | ซ่อน Role Switcher ในโหมดปกติ | 1. เข้าลิงก์ปกติ `http://localhost:3001/` | แถบสีส้มสำหรับกดสลับสิทธิ์จำลองถูกซ่อนไว้ 100% ไม่ปรากฏให้ผู้ใช้ทั่วไปเห็น | 🟢 Pass |
| **TC-NAV-05** | แสดง Role Switcher โหมด Debug | 1. เข้าลิงก์ `http://localhost:3001/?debug=true` | แถบสีส้ม Quick Role Switcher โชว์ขึ้นมาบนสุด ให้ผู้พัฒนากดทดสอบสิทธิ์ได้ | 🟢 Pass |

## 🗄️ มิติที่ 3: การเชื่อมต่อฐานข้อมูล Cloud Firestore (Cloud Database Sync: TC-DATA)

| รหัสเทสเคส | ชื่อรายการทดสอบ | ขั้นตอนการทดสอบ (Test Steps) | ผลลัพธ์ที่คาดหวัง (Expected Results) | สถานะ |
| :--- | :--- | :--- | :--- | :---: |
| **TC-DATA-01** | ดึงข้อมูลจาก Collection `flooddata` | 1. สร้าง Document ใน Firestore ชื่อ `cha-uat` ตั้งค่า `floodChance: 53`<br>2. รีเฟรชหน้าเว็บแอปพลิเคชัน | โอกาสท่วมของการ์ดอำเภอชะอวดเปลี่ยนจากค่าม็อกเป็น 53% ตรงตาม Firestore | 🟢 Pass |
| **TC-DATA-02** | ดึงข้อมูลจาก Collection `location` | 1. สร้าง Document ใน Collection `location`<br>2. ตรวจสอบการโหลดข้อมูล | ระบบดึงข้อมูลจากทั้ง `flooddata` และ `location` มาหลอมรวมกันแสดงผลในหน้าแรก | 🟢 Pass |
| **TC-DATA-03** | อัปเดตค่าความเสี่ยงเรียลไทม์ | 1. เปิดหน้าเว็บทิ้งไว้ แก้ค่า `floodChance: 153` ใน Firebase Console | หน้าเว็บรับรู้ข้อมูลใหม่และเปลี่ยนตัวเลขความเสี่ยงทันทีเมื่อมีการรีเฟรชหรือรีโหลดคอมโพเนนต์ | 🟢 Pass |
| **TC-DATA-04** | รองรับสถานที่ใหม่ (Dynamic Addition) | 1. กด Add Document ใส่ชื่อหมู่บ้านใหม่ที่ไม่เคยมีในระบบลงใน Collection `location` | หน้าเว็บสร้างการ์ดสถานที่ใหม่เพิ่มต่อท้ายในกริดหน้าแรกให้อัตโนมัติ ไม่เกิดข้อผิดพลาด | 🟢 Pass |
